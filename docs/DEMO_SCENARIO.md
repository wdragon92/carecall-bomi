# 발표용 시연 대본 (돌봄콜 AI)

준비: `python run.py` → http://127.0.0.1:8080 접속. 헤더에 `실시간 연동 상담 중` 표시 확인.
(권장) 브라우저 창을 넓게 — 우측 특이사항/복지 패널이 함께 보입니다. 음성은 🔊 ON 상태로.

## 30초 한 줄 소개
"한국 복지는 알아야 받는 '신청주의'입니다. 돌봄콜 AI는 어르신께 먼저 안부를 여쭙고, 대화에서 건강·정서·복지 신호를
실시간으로 읽어 자격 맞는 복지를 알려주고 사기까지 예방합니다. 전부 국내 CLOVA로 처리되는 소버린 AI입니다."

## 시연 플로우 (약 3분)

1. **첫 화면 — 선인사**
   - 접속하면 봄이가 시간대에 맞춰 먼저 인사("어르신, 안녕하세요…"). 음성으로도 자동 재생됨.

2. **건강 신호 → 특이사항 실시간 추출** (핵심)
   - 입력: `요즘 밤에 잠을 통 못 자요.`
   - 봄이가 공감하며 짧게 되물음. 동시에 우측 패널에 **건강 카드(노랑)** 가 스르륵 추가됨.

3. **정서 + 복지 니즈 → 복지 자동 매칭**
   - 입력: `혼자 살다 보니 외롭고, 생활비도 빠듯해요.`
   - **정서 카드** 추가 + **복지 탭**에 기초연금·생계급여·노인맞춤돌봄서비스 등이 뜸.
   - 봄이가 자연스럽게 복지 안내를 대화에 녹임.

4. **사진 첨부 → OCR + 사기 판별**
   - 📎로 의심 문자(스미싱) 캡처를 첨부.
   - 봄이가 내용을 쉬운 말로 설명하고 "링크 누르지 마세요"라고 경고 + **사기_노출 카드(빨강)**.

5. **음성 대화 (STT/TTS)**
   - 🎤을 **한 번 눌러** 말하고, 끝나면 **다시 눌러** 전송 → 인식된 문장이 내 말풍선으로. 봄이 답변이 음성으로 재생.

6. **긴급 시나리오 (선택, 가드레일 강조)**
   - 입력: `사는 게 다 부질없고 그만 살고 싶어요.` → 상단 **빨간 긴급 배너** + "119/보호자 연결 권고".
   - 포인트: AI가 해결하려 들지 않고 **사람에게 연결**을 권함(진단·치료 아님).

7. **상담 마치기 → 요약 리포트**
   - `상담 마치기` → 요약·특이사항·후속 권고·안내한 복지가 담긴 리포트 모달.

## 어필 포인트
- **실시간성**: 대화하는 즉시 패널이 갱신(WebSocket 스트리밍).
- **정확성/안전**: 복지 금액·자격은 `welfare.json` 근거로만, 진단 금지, 긴급은 사람에게.
- **접근성**: 큰 글씨·큰 버튼·고대비·음성, 모바일 탭 UI.
- **소버린 AI**: STT·LLM·TTS·OCR 전부 국내 NCP CLOVA. 데이터 국내 처리.
- **견고함**: 특정 API 실패 시 해당 기능만 mock 폴백 → 시연이 끊기지 않음.

## 돌발 대비
- 응답이 느리면: 네트워크/모델 지연. 잠시 기다리면 스트리밍으로 나옴.
- 마이크 무반응: localhost/https에서만 동작. 브라우저 마이크 권한 허용 필요.
- 특정 기능이 'mock'으로 표시: 헤더 상태 확인, 해당 키/도메인 점검(README 실연동 표).
