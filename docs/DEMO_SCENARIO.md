# 발표용 시연 대본 (돌봄콜 AI — 똑똑한 돌봄 친구 보미)

준비: `python build_index.py --source fixtures`(발표 전날 1회) → `python run.py` → http://127.0.0.1:8080 접속.
헤더에 `보미가 듣고 있어요 (실시간 연동)` 확인. `/health`에서 `rag.loaded:true`도 확인.
(권장) 브라우저 창을 넓게 — 우측 특이사항/복지 패널이 함께 보입니다. 음성은 🔊 ON 상태로.

## 30초 한 줄 소개
"한국 복지는 알아야 받는 '신청주의'입니다. 저희는 집집마다 AI 스피커를 놓듯 어르신 곁에 **똑똑한 돌봄 친구 '보미'** 를
둡니다. 매일 안부를 살피고, 대화에서 건강·정서·복지 신호를 실시간으로 읽고, **공식 복지 자료를 검색(RAG)해 근거 있는
안내**를 하고, 사기까지 예방합니다. 검색 인덱스는 서버 안 FAISS — 데이터가 밖으로 나가지 않는 소버린 AI입니다."

## 시연 플로우 — 킬러 3+1 (약 4분)

1. **첫 화면 — 비서의 선인사**
   - 접속하면 보미가 시간대에 맞춰 먼저 인사(전화가 아니라 늘 곁에 있는 톤). 음성 자동 재생.

2. **킬러 ① 의미로 찾는다 (RAG 하이브리드 검색)**
   - 입력: `무릎이 아파서 장 보러 가기가 힘들어요`
   - '복지'라는 단어가 없어도 → 보미가 공감 + **노인맞춤돌봄서비스**를 대화로 풀어 설명하고,
     이어서 **📌 정보 카드**(파란 테두리)가 뜸: 대상·지원·신청처·**정보 기준일**.
   - 멘트: "키워드가 아니라 **의미**로 찾습니다. 그리고 **금액·신청처 같은 사실은 LLM이 아니라
     코드가 공식 자료 필드에서 그대로 넣습니다** — 수치 환각이 원천 차단됩니다. 카드 하단 기준일이 그 증거입니다."
   - 🔊이면 카드가 통째로 낭독되지 않고 "정보 카드로 정리해 드렸어요" 한 문장만 읽음(어르신 UX).

3. **킬러 ② 판정하고 신청까지** *(P3 구현 후)*
   - 입력: `기초연금 나도 받을 수 있나?`
   - 보미가 나이·가구형태를 대화로 여쭙고 → **룰엔진(코드)** 이 "가능성" 판정 + 서류 체크리스트/신청처 패키지.
   - 멘트: "판정은 LLM이 아니라 **결정론적 룰**입니다. 고시 상수는 사람이 확인해 넣고, '확정'은 구청 조사의 몫이라
     늘 '가능성'으로만 말합니다."

4. **킬러 ③ 지어내지 않는다 (거부 + 연계)**
   - 입력: `로또 당첨 번호 좀 알려줘` (또는 `요즘 대통령이 누구더라`)
   - 보미는 복지 자료 밖 질문에 수치를 지어내지 않고, 필요하면 129·주민센터로 정중히 연계.
   - 멘트: "의미 점수와 어휘 증거 **2단 게이트**로 자료 밖 질문을 거릅니다.
     평가셋 실측: **Hit@2 100% / 인도메인 오거부 0 / 문서밖 거부 83%** (data/eval_results.json)."

5. **(+1) 사진 → OCR + 사기 판별**
   - 📎로 의심 문자(스미싱) 캡처 첨부 → 쉬운 말 설명 + "링크 누르지 마세요" 경고 + **사기_노출 카드(빨강)**.

6. **긴급 가드레일 (선택)**
   - 입력: `사는 게 다 부질없고 그만 살고 싶어요.` → 상단 **빨간 긴급 배너** + 사람 연결(109) 권고.
   - 포인트: AI가 해결하려 들지 않고 **사람에게 연결**(진단·치료 아님).

7. **이야기 마치기 → 돌봄 요약 리포트**
   - 요약·특이사항·후속 권고·**안내한 복지(기준일 포함)** 리포트 모달 — 보호자/담당자 전달용.

## 어필 포인트
- **상시 비서 피벗**: 걸려오는 안부 전화가 아니라 집에 늘 있는 돌봄 비서 — 안부는 스치듯, 도움은 근거 있게.
- **RAG 정확도 3계층**: T1(제도·신청법은 확정 안내) / T2(수치는 코드가 카드로 삽입 + 기준일 표시) / T3(개인 확정은 129·주민센터 연계).
- **실시간성**: 대화 즉시 특이사항·복지 패널 갱신(WebSocket).
- **소버린 AI**: STT·LLM·TTS·OCR·임베딩 전부 NCP CLOVA + 검색은 서버 내 FAISS(데이터 외부 유출 없음).
- **견고함**: 인덱스는 로컬이라 외부 API 장애와 무관. provider 실패는 개별 mock 폴백 — 시연이 끊기지 않음.
- **신선도**: 주 1회 배치 + 변경분만 재임베딩(비용≈0) + 답변 직전 상세조회(P0 후) — "정보 기준일"로 투명하게.

## 돌발 대비
- 응답이 느리면: 네트워크/모델 지연. RAG 검색 자체는 로컬(ms)이라 병목 아님.
- 인덱스 미로드(`rag.loaded:false`): `python build_index.py --source fixtures` 후 `POST /api/rag/reload` — 재시작 불필요.
- 마이크 무반응: localhost/https에서만 동작. 브라우저 마이크 권한 허용 필요.
- 특정 기능 'mock' 표시: 헤더 상태 확인, 해당 키/도메인 점검(README 실연동 표).
